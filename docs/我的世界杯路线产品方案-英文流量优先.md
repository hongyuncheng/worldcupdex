# My World Cup Route 产品方案（英文流量优先）

日期：2026-05-27  
项目：WorldCupDex  
优先级建议：P1，建议作为第 13 项独立功能推进

---

## 1. 产品定位

### 一句话定位

`My World Cup Route` 是面向英文世界杯球迷的个人化球队追踪工具，帮助用户围绕自己支持的球队查看下一场比赛、本地开球时间、小组赛路线、出线槽位、潜在淘汰赛路径、日历提醒和预测入口。

它要解决的问题不是“给用户看完整 104 场比赛”，而是回答：

- When is my team’s next World Cup match?
- What time is it in my timezone?
- Who does my team play next?
- What group is my team in?
- What happens if my team finishes first or second?
- Which matches affect my team’s route?
- Can I add my team’s fixtures to my calendar?

### 站点定位升级

当前 WorldCupDex 更像：

> World Cup encyclopedia + schedule + prediction tool.

加入该功能后，可以升级为：

> A personal World Cup companion for fans tracking their team’s journey.

这对英文流量尤其重要，因为英文用户更可能通过以下搜索进入：

- `Argentina World Cup 2026 route`
- `Brazil path to final 2026`
- `USA World Cup next match`
- `England World Cup schedule local time`
- `Mexico World Cup group path`
- `France World Cup knockout route`

## 2. 目标用户与核心需求

### 目标用户

优先服务英文流量，因此核心用户画像按英语搜索需求定义：

1. Team-first fans  
   只关心自己支持的球队，例如 Argentina、Brazil、England、USA、Mexico、France。

2. Host-country fans  
   美国、加拿大、墨西哥用户更关注本地时间、主办城市、现场观赛和主队路线。

3. Star-driven fans  
   围绕球星关注球队，例如 Messi/Argentina、Mbappe/France、Bellingham/England。

4. Casual tournament followers  
   不会研究完整赛程，但想知道“我队下一场”和“赢了之后会打谁”。

### 核心需求

高频需求：

- 下一场比赛：next match
- 本地时间：local kickoff time
- 小组赛完整路线：group stage fixtures
- 小组对手：group opponents
- 关键同组比赛：matches that affect my team
- 出线路径：if finish 1st / 2nd
- 潜在淘汰赛路径：possible knockout path
- 加入日历：add to calendar
- 预测比赛：predict match

低频但有传播价值的需求：

- path to final
- road to final image/card
- bracket scenario
- shareable route card

## 3. MVP 范围

### MVP 必做

第一版目标是做出可被 Google 收录、能被球迷真实使用、能从现有数据生成的路线页。

必做功能：

1. 新增球队路线页：
   - `/teams/{team}/world-cup-2026-route`
   - `/zh/teams/{team}/world-cup-2026-route`
   - `/es/teams/{team}/world-cup-2026-route`

2. 页面展示：
   - team hero
   - next match
   - local kickoff time
   - countdown
   - venue and host city
   - group stage timeline
   - group opponents
   - related group matches
   - if finish 1st / 2nd knockout slot
   - add to calendar
   - predict match
   - internal links

3. SEO：
   - title
   - description
   - canonical
   - hreflang
   - OG
   - sitemap
   - prerender
   - `SportsTeam` / `SportsEvent` / `BreadcrumbList` JSON-LD

4. 现有页面入口：
   - team detail page
   - team schedule page
   - homepage module

5. 本地关注：
   - 复用现有 favorite/follow 基础
   - 第一版可只用 `localStorage`
   - 支持 primary team

### MVP 暂不做

以下功能可以后续做，不建议第一版阻塞上线：

- 实时积分榜
- 出线概率
- AI 自动晋级预测
- 胜平负模拟器
- 登录账号同步
- push notification
- 复杂 bracket 可视化
- 可分享长图

原因：第一版核心是验证英文 SEO 长尾和工具需求，不应被动态数据和复杂模拟拖慢。

## 4. 页面结构

## 4.1 首页模块

### 模块名称

英文优先，建议使用：

- `Track Your Team`
- `My World Cup Route`
- `Follow Your Team’s Path`

推荐主标题：

> Track Your Team’s World Cup Route

### 位置

建议放在首页倒计时之后、核心导航卡片之前，成为准首屏动作。

### 结构草图

```text
Track Your Team’s World Cup Route
Follow your team from the group stage to the knockout rounds.

[ Search or select a team...         ▼ ]

Popular teams:
Argentina  Brazil  England  USA  Mexico  France
```

用户选择球队后：

```text
Argentina Route

Next match
Argentina vs Algeria
Wed, Jun 17 · 4:00 PM your time
SoFi Stadium · Inglewood

[View Full Route] [Add to Calendar]
```

首页模块只做一件事：让用户选择自己的球队，并进入路线页。

不要在首页展示过多路线细节，否则会压缩核心入口效率。

### 推荐 URL

英文 SEO 优先，推荐：

```text
/teams/{team}/world-cup-2026-route
```

```text
/teams/{team}/route
```

原因：短 URL 产品内好看，但 SEO 意图不如 `world-cup-2026-route` 明确。

## 5. 数据结构

## 5.1 现有数据

### teams 数据

需要字段：

```ts
type Team = {
  id: string
  name: string
  nameZh?: string
  nameEs?: string
  flag: string
  group: string
  fifaRanking?: number
  confederation?: string
}
```

### matches 数据

需要字段：

```ts
type Match = {
  id: string
  stage: 'group' | 'round-of-16' | 'quarter-final' | 'semi-final' | 'final'
  group?: string
  homeTeam: TeamRef
  awayTeam: TeamRef
  kickoffUtc: string
  venue: string
  city: string
  status: 'scheduled' | 'live' | 'finished'
  score?: {
    home: number
    away: number
  }
}
```

## 5.2 新增 knockout slots 数据

新增文件建议：

```text
data/knockout-slots.json
```

作用：描述官方淘汰赛槽位，而不是预测具体对手。

示例：

```json
[
  {
    "round": "Round of 16",
    "matchId": "r16-1",
    "slotA": "A1",
    "slotB": "B2",
    "date": "2026-07-04T00:00:00Z",
    "venue": "TBD",
    "city": "TBD"
  },
  {
    "round": "Round of 16",
    "matchId": "r16-2",
    "slotA": "C1",
    "slotB": "D2",
    "date": "2026-07-04T00:00:00Z",
    "venue": "TBD",
    "city": "TBD"
  }
]
```

注意：这里必须核对官方 2026 bracket slot，不要凭记忆写死。

## 5.3 派生函数

建议新增：

```text
composables/useTeamRoute.ts
```

或：

```text
utils/teamRoute.ts
```

核心函数：

```ts
getTeamRoute(teamId: string): TeamRoute
```

返回结构：

```ts
type TeamRoute = {
  team: Team
  group: string
  groupTeams: Team[]
  nextMatch: Match | null
  groupMatches: Match[]
  relatedGroupMatches: Match[]
  qualificationPaths: QualificationPath[]
  calendarEvents: CalendarEvent[]
  seo: TeamRouteSeo
}

type QualificationPath = {
  finish: '1st' | '2nd'
  slot: string
  roundOf16Match: KnockoutSlot | null
  label: string
}

type TeamRouteSeo = {
  title: string
  description: string
  canonical: string
  ogImage?: string
}
```

## 5.4 本地关注状态

第一版建议继续使用浏览器本地状态。

```ts
type FollowedTeamState = {
  teamIds: string[]
  primaryTeamId?: string
  timezone?: string
}
```

设计原则：

- `teamIds`：用户关注的全部球队。
- `primaryTeamId`：我的世界杯路线主队。
- 如果用户只关注 1 个队，自动设置为 primary team。
- 如果用户关注多个队，首页只突出 primary team，其它队做次级入口。

## 6. SEO 方案

## 6.1 第一阶段 URL 矩阵

主模板：

```text
/teams/{team}/world-cup-2026-route
```

示例：

```text
/teams/argentina/world-cup-2026-route
/teams/brazil/world-cup-2026-route
/teams/france/world-cup-2026-route
/teams/england/world-cup-2026-route
/teams/united-states/world-cup-2026-route
/teams/mexico/world-cup-2026-route
```

多语言：

```text
/zh/teams/{team}/world-cup-2026-route
/es/teams/{team}/world-cup-2026-route
```

英文优先意味着：

- 英文 URL 是默认 canonical 主路径。
- 中文、西语作为 hreflang alternate。
- 英文文案优先打磨。
- 中文、西语可以在第一版保持可读，但后续重点页面需要人工优化。

## 6.2 Title 模板

英文：

```text
{Team} World Cup 2026 Route: Fixtures, Group Path & Knockout Scenarios | WorldCupDex
```

中文：

```text
{TeamZh} 2026 世界杯路线：赛程、小组出线与淘汰赛路径 | WorldCupDex
```

西语：

```text
Ruta de {Team} en el Mundial 2026: partidos, grupo y posibles cruces | WorldCupDex
```

## 6.3 Description 模板

英文：

```text
Track {Team}’s 2026 World Cup route with fixtures, local kickoff times, venues, group-stage opponents, calendar export and possible knockout paths.
```

中文：

```text
追踪 {TeamZh} 2026 世界杯路线，包括小组赛赛程、本地开球时间、比赛球场、出线槽位、日历导出和潜在淘汰赛路径。
```

西语：

```text
Sigue la ruta de {Team} en el Mundial 2026 con partidos, horarios locales, sedes, rivales de grupo, calendario y posibles cruces.
```

## 6.4 页面正文 SEO 重点

页面不要堆 AI 文案，重点放在工具数据：

- fixtures
- local kickoff times
- venues
- group opponents
- knockout slots
- calendar export
- predictions

SEO block 可以控制在 1 段短文：

```text
This page helps {Team} fans track the 2026 World Cup route without scanning the full tournament schedule. It combines {Team}'s group-stage fixtures, local kickoff times, venue information, calendar tools and possible knockout slots in one place.
```

## 10. 风险与注意事项

### 风险 1：淘汰赛槽位准确性

必须核对官方 bracket slot。不要凭印象写死。

### 风险 2：页面矩阵过快铺开

第一版只做 route 页。`next-match`、`path-to-final`、`qualification-scenarios` 后续再扩展。

### 风险 3：西语质量

英文优先可以接受，但西语页面不能大量混英文。至少 title、description、核心 CTA 应完成本地化。

### 风险 4：Follow 和 Route 混乱

必须区分：

- Followed teams：关注的多个队。
- Primary route：我的主路线队伍。

### 风险 5：比赛未开始前没有真实出线形势

页面文案要诚实：

```text
Knockout slots are based on official bracket positions. Live qualification status will update after matches begin.
```

## 13. 最终建议

优先英文流量时，该功能应被包装为：

> Track your team’s 2026 World Cup route.

而不是中文语境里的“收藏球队”或“我的关注”。

产品上，它是个人化赛事助手；SEO 上，它是球队长尾路线页矩阵；商业化上，它是比赛前情绪峰值入口。

最推荐的第一步是：

1. 先实现 `/teams/{team}/world-cup-2026-route`。
2. 再把首页、球队详情页、球队赛程页入口打通。
3. 最后做 `primary team` 的个人化首页模块。
