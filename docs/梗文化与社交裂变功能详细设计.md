# WorldCupDex “梗”文化与社交裂变功能详细设计文档

## 概述
为了最大化 WorldCupDex 的社交传播属性，弥补无实时比分数据的短板，本项目将引入三个极具娱乐性和社交货币属性的核心功能：**赛后“背锅/大腿”票选**、**“毒奶”预测卡**、**球员表情包（Meme）生成器**。

本文档详细定义了这三个功能的产品逻辑、UI/UX 交互及技术实现方案。

---

## 模块一：赛后“背锅侠 / 真大腿”票选系统（Post-Match Awards）

### 1. 核心定位
将球迷赛后的情绪发泄需求转化为网站互动与裂变流量。利用极具反差感的标签（如“叹息之墙”、“拙劣的表演”）激发用户的表达欲。

### 2. 产品设计与交互流程
*   **入口**：
    *   `/schedule`（赛程页）：在已结束（`status: 'FINISHED'`）的比赛卡片下方增加醒目的【赛后评级】按钮。
    *   `/matches/[id]`（比赛详情页）：页面核心区域展示投票模块。
*   **投票界面**：
    *   分两个 Tab：`谁是真大腿 (MVP)` 和 `谁来背大锅 (Scapegoat)`。
    *   每个 Tab 下展示 3-5 名本场关键球员的头像（或默认国家队队徽+球员名）。
    *   用户点击某个球员进行投票。
*   **结果与分享页**：
    *   投票后立即显示当前投票比例（利用动画填充进度条）。
    *   自动生成一张极具视觉冲击力的【颁奖海报】。
    *   **海报文案示例**：
        *   大腿版：`“实至名归！梅西以 85% 选票当选本场真大腿！[网站水印]”`
        *   背锅版：`“众望所归！卢卡库荣膺本场叹息之墙，超过 50,000 名球迷为他点赞！[网站水印]”`
    *   触发“Share to Unlock”机制（可选）：分享后解锁查看详细得票数据。

### 3. 技术实现方案
*   **数据存储 (Cloudflare KV)**：
    *   复用并扩展目前的 `vote.post.ts` 和 `vote.get.ts`。
    *   KV 键名设计：
        *   `match:${matchId}:mvp:${playerId}` -> 存储得票数 (int)
        *   `match:${matchId}:scapegoat:${playerId}` -> 存储得票数 (int)
*   **防刷票机制**：
    *   基于 IP hash + 浏览器 localStorage（标记 `voted_match_${matchId}`），限制同一台设备单场比赛只能投一次。
*   **候选人数据来源**：
    *   **初期简易版**：读取 JSON 里的球队阵容（如果有），或硬编码几名球星，若无具体球员则直接提供“教练”、“裁判”、“VAR”、“整体防线”等通用选项。
    *   **进阶版**：通过 GitHub Actions 赛后抓取核心球员名单写入临时 JSON。

## 模块二：世界杯专属“毒奶（Jinx）”预测卡

### 1. 核心定位
利用足球圈流行的“反向预测/毒奶”文化，制造更具传播性和话题性的预测卡片，期待比赛爆冷后形成“挖坟式”的二次裂变。

### 2. 产品设计与交互流程
*   **入口与交互**：
    *   在现有的 `/predict/[id]` 和 `/predict/champion` 页面。
    *   在常规的“生成预测卡”按钮旁边，增加一个具有魔法/暗黑风格的 Switch 开关或按钮：【🔮 开启毒奶模式】。
*   **UI 突变**：
    *   开启后，页面背景、卡片样式瞬间切换（例如切换为紫色/幽绿色调，增加法阵/魔法闪烁特效）。
*   **海报生成与文案**：
    *   **常规文案**：“我预测阿根廷将 2:0 战胜法国。”
    *   **毒奶文案模板**（随机抽取）：
        *   `“我单方面宣布，阿根廷提前锁定冠军！（毒奶施法中 🪄）”`
        *   `“稳了！法国这波绝对不可能输，我话放在这了！🥛”`
        *   `“这局要是巴西不赢，我当场把键盘吃了！🚩”`
    *   **卡片视觉**：在卡片显著位置盖上“东方神秘力量”、“毒奶认证”等显眼的印章。

### 3. 技术实现方案
*   **纯前端改造**：
    *   无需后端接口修改。
    *   在 `PredictionCard.vue` 中新增 `isJinxMode` 属性。
    *   根据该属性，动态切换 CSS class（调用 DaisyUI 的其他主题配色或自定义的 `.jinx-theme`）。
    *   在文本渲染逻辑中，加入毒奶文案字典（中/英/西三语），随机抽取渲染。
    *   利用 `html2canvas-pro` 正常截图即可。

## 模块三：DIY 球员表情包生成器（Meme Generator）

### 1. 核心定位
提供一个极低门槛的工具，让球迷能够迅速将世界杯名场面转化为带有本站水印的表情包，利用社交网络的自发传播获取免费流量。

### 2. 产品设计与交互流程
*   **入口**：
    *   主导航新增一个 Tab：【Meme / 梗图】（路径 `/meme`）。
    *   或在 `/wiki`、`/teams/[id]` 球员列表旁增加快捷入口。
*   **交互界面**：
    *   **素材库**：左侧/顶部提供预设的素材图（本站抓取的球星大头照、世界杯经典名场面如红牌、假摔等）。支持用户上传本地图片。
    *   **编辑区**：居中展示图片。
    *   **文本输入**：提供“上部文字”和“下部文字”两个输入框。
*   **渲染效果**：
    *   文字强制使用经典 Meme 样式：白色字体，粗黑描边（Impact 字体），全部大写，自动缩放以适应图片宽度。
    *   图片右下角自动且强制嵌入半透明的 `WorldCupDex.org` 网址水印。
*   **输出与分享**：
    *   点击【生成梗图】，直接弹出保存/分享框。
    *   不涉及复杂的业务逻辑，即玩即走。

### 3. 技术实现方案
*   **前端渲染 (Canvas)**：
    *   使用原生 HTML5 Canvas 或现有的 `html2canvas-pro` 将图片、文字、水印合并绘制。
    *   CSS 文本描边技巧：`-webkit-text-stroke: 2px black; color: white; font-family: Impact, sans-serif;`
*   **素材准备**：
    *   需要在 `public/images/memes/` 下预置 10-20 张高清晰度的无版权争议图片或我们抓取的头像。
*   **无后端依赖**：
    *   整个生成过程完全在客户端完成，0 服务器资源消耗。

## 实施优先级建议

1.  **优先级最高（ROI 最大，开发最快）：模块二（毒奶预测卡）**。纯前端改动，直接复用现有预测流程，能立刻增加现有预测卡的分享率。
2.  **优先级次之（互动深度最强）：模块一（背锅侠票选）**。需要前后端联动（CF KV），但能极大增加赛后的用户回访率。
3.  **优先级最后（独立工具）：模块三（Meme 生成器）**。作为一个独立的引流工具，可以在基础功能完善后，作为赛前预热的独立活动推出。
