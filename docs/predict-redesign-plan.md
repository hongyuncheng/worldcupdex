# 预测竞猜模块改造计划：从积分制到预测卡片制

## 一、改造背景

### 当前问题

| 问题 | 详情 |
|------|------|
| 积分无消耗场景 | 积分只能看数字，无兑换、无实际用途 |
| 排行榜为假数据 | 5条 Mock 数据，无真实用户参与 |
| 需要登录系统 | 增加开发成本和用户门槛 |
| 页面过重 | 6个Tab、积分规则、排行榜、历史记录等模块臃肿 |
| 提交流程未完成 | 详情页提交按钮无事件处理 |
| 传播性弱 | 积分系统无法激励分享 |

### 改造目标

**核心定位**：将"积分游戏"改造为"预测卡片生成器"——用户提交预测后立即生成精美的可分享卡片，驱动社交传播。

**设计原则**：
- 零登录门槛（无需账号系统）
- 轻量交互（最少步骤完成预测）
- 强传播（每次预测 = 一次社交曝光）
- 复用已有技术（html2canvas-pro + SharePanel）

## 二、功能规划

### 2.1 保留的玩法

| 玩法 | 说明 | 优先级 |
|------|------|--------|
| **单场预测卡** | 选择一场比赛 → 预测胜负/比分 → 生成卡片 | P0 |
| **冠军竞猜卡** | 选择心中的冠军球队 → 生成卡片 | P0 |
| **赛后战绩卡** | 赛后自动对比预测与实际结果 → 生成验证卡片 | P1（赛中上线） |

### 2.2 移除的模块

| 模块 | 原位置 | 原因 |
|------|--------|------|
| 积分规则 Tab | 主页侧栏 + Tab | 不再需要积分 |
| 排行榜 | 主页侧栏 | 无真实数据，无登录系统 |
| 我的预测概览（积分相关） | 主页侧栏 | 改为"我的预测卡集" |
| 历史战绩 Tab | Tab导航 | 简化为本地预测记录 |
| 倒计时卡片 | Hero区域 | 简化页面 |
| 计分规则详情 | Tab页面 | 不再计分 |
| useScoring.ts | composables | 积分逻辑不再需要 |

## 六、数据流设计

### 6.1 单场预测流程

```
用户进入 /predict/[id]
    │
    ▼
加载比赛信息（useMatches）
    │
    ▼
用户选择：胜/平/负 + 比分（可选）
    │
    ▼
点击"生成预测卡片"
    │
    ├── 1. 保存到 localStorage (usePredictions)
    │      key: wcd_predictions
    │      value: { [matchId]: { result, score, timestamp } }
    │
    ├── 2. 显示 PredictionCard 组件（渲染卡片）
    │
    └── 3. 显示 SharePanel（分享/下载）
              │
              ├── html2canvas → PNG Blob
              ├── 移动端 → Web Share API
              └── PC端 → 下载 / 社交平台链接
```

### 6.2 冠军竞猜流程

```
用户进入 /predict/champion
    │
    ▼
展示 48 支球队（国旗+队名网格）
    │
    ▼
用户点击选择一支球队
    │
    ▼
点击"生成冠军预测卡"
    │
    ├── 1. 保存到 localStorage
    │      key: wcd_champion_prediction
    │      value: { teamId, teamName, timestamp }
    │
    ├── 2. 显示 ChampionCard 组件
    │
    └── 3. 显示 SharePanel
```

### 6.3 本地存储设计

```typescript
// localStorage 键值规划
{
  // 单场预测记录
  "wcd_predictions": {
    [matchId: number]: {
      matchId: number
      result: 'HOME_WIN' | 'AWAY_WIN' | 'DRAW'
      score?: { home: number; away: number }
      timestamp: number
    }
  },

  // 冠军竞猜记录
  "wcd_champion_prediction": {
    teamId: string
    teamName: string
    teamNameEn: string
    timestamp: number
  },

  // 已生成卡片记录（用于"我的预测卡集"展示）
  "wcd_prediction_cards": [
    {
      type: 'match' | 'champion'
      matchId?: number
      teamId?: string
      timestamp: number
      // 不存图片，需要时重新渲染
    }
  ]
}
```

## 八、风格规范

### 配色体系

| 用途 | 颜色 | 说明 |
|------|------|------|
| 页面背景 | `#000F49` → `#1A237E` | 深蓝渐变 |
| 卡片背景 | 球队主题色（team-colors.ts） | 每张卡片独特 |
| 强调色 | `#FFD700` | 金色，用于预测结果高亮 |
| 正确 | `#22C55E` | 绿色，赛后验证用 |
| 错误 | `#EF4444` | 红色，赛后验证用 |
| 文字 | `#FFFFFF` / `rgba(255,255,255,0.7)` | 白色系 |

### 字体

- 标题：`Montserrat`, sans-serif (Bold 700-800)
- 正文：`Inter`, sans-serif (Regular 400, Medium 500)

### 卡片出图规格

- 渲染尺寸：400×520px（CSS像素）
- 导出尺寸：800×1040px（2x scale）
- 格式：PNG（透明背景）
- 文件命名：`worldcupdex-prediction-{matchId}-{timestamp}.png`
